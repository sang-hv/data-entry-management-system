# M3.1 — Task Done Flag (bỏ %, thêm ghi chú per-order-task)

> **Status:** 🔧 In progress
> **Ngày:** 02/06/2026
> **Tiền đề:** chạy trước M4 (Batches & Apply Ratio).
> **Liên quan:** sửa logic task đã làm ở M3 (`docs/plans/M3-orders-and-tasks.md`).

## Bối cảnh

M3 cài task workflow với `OrderTask.progressPct` (0..100). Trạng thái đơn + `Order.progressPct`
auto-derive từ trung bình % các task. Khách yêu cầu đơn giản hóa: khi đơn pick task từ thư viện,
mỗi task chỉ cần **tick xong / chưa xong** (bỏ %), và có **ghi chú riêng cho task gắn với đơn**.

## Quyết định (Hướng A — đã chốt với user)

- `OrderTask` dùng boolean `done` thay cho `progressPct`. Giữ lại `Order.progressPct` nhưng đổi
  ý nghĩa thành **% công đoạn đã xong** = `round(số task done / tổng task × 100)`.
- Ghi chú riêng = `OrderTask.notes` (đã có sẵn từ M3) — ghi chú gắn với task trong đúng 1 đơn,
  khác `Task.description` (master) và `Order.notes` (cả đơn).

## Thay đổi schema

`OrderTask`:
- ❌ Bỏ `progressPct Int @default(0)`
- ❌ Bỏ `startedAt DateTime?`
- ✅ Thêm `done Boolean @default(false)`
- ✅ Giữ `completedAt DateTime?` (set khi `done=true`, clear khi `done=false`), `notes String?`

`Order.progressPct` giữ kiểu `Int` (0..100), đổi ngữ nghĩa (cache % task hoàn thành).

Migration: drop cột `progressPct`, `startedAt` của `OrderTask`; add `done`. DB chưa có Order data
thật nên không lo mất dữ liệu.

## Logic (modules)

- `order.progress.ts`:
  - `OrderTaskMinimal`: `{ progressPct }` → `{ done: boolean }`
  - `computeOrderStatus`: COMPLETED khi mọi task `done`; DRAFT khi rỗng; còn lại ACTIVE.
  - `computeOrderProgress`: `round(doneCount / total × 100)`.
- `recompute.ts`: `select { progressPct }` → `select { done }`. Giữ logic set/clear `actualAt`.

## Action layer

- `updateOrderTaskProgress` → đổi tên **`setOrderTaskDone`**. Input `{ orderTaskId, done, notes? }`.
  Bỏ `startedAt`; `completedAt` set/clear theo `done`. History note: `→ Hoàn thành` / `→ Chưa xong`.
- `pickTasksForOrder`: bỏ `progressPct: 0` khi tạo (mặc định `done=false`).
- `setOrderTasks`: item bỏ `progressPct`, thêm `done` (default false); bỏ `startedAt`.

## API

`server/api/order-tasks/[id]/progress.patch.ts` → `done.patch.ts`, body `{ done, notes }`.

## UI

- `OrderTaskStepper.vue`: bỏ thanh % + badge %. Mỗi task có nút tick (✓ xanh khi done, xám khi chưa).
  Badge 2 trạng thái: Hoàn thành / Chưa xong. Hộp thoại sửa: toggle done + ô Ghi chú.
- `OrderProgressBar.vue`: giữ nguyên (vẫn nhận `value` 0..100 = % task xong).
- `pages/orders/[id].vue`: `updateTaskProgress` → `updateTaskDone`; payload bỏ `progressPct`.
- `pages/orders/index.vue`: giữ nguyên.
